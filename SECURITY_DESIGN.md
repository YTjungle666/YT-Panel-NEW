# YT-Panel 安全设计文档

## 加密方案设计

### 为什么不用硬编码密钥？

**硬编码密钥的问题：**
- 密钥写在代码里，任何人都能看到
- 代码泄露 = 密钥泄露
- 无法更换密钥

**我们的方案：**
服务器每天动态生成密钥，前端启动时获取。

### 工作流程

```
用户打开页面
    ↓
前端请求 /api/crypto-key
    ↓
后端返回当天密钥（基于日期生成）
    ↓
前端用密钥加密/解密本地存储
    ↓
用户登出 → 清除内存中的密钥
```

### 降级机制

如果后端是新版本但前端是旧版本，或者密钥获取失败：
- 自动降级为明文存储
- 不报错，不影响使用
- 兼容旧数据和自托管场景

### Docker/PVE 兼容性

- 无需配置环境变量
- 开箱即用
- 支持 LXC 容器
