# YT-Panel-NEW 安全审计报告

## 审计日期: 2026-04-04
## 审计范围: Rust 后端代码 + 前端构建产物

---

## 1. 高风险问题

### 1.1 SQL 注入风险 (中等)
**位置**: `backend/src/main.rs:load_user_by()` 函数
**问题**: 使用 format! 动态构建 SQL 查询，虽然当前调用是安全的，但存在潜在风险
**代码**:
```rust
let query = format!("SELECT ... FROM user WHERE {} = ?", field);
```
**建议**: 使用枚举类型限制 field 参数，避免字符串拼接

### 1.2 密码哈希强度
**位置**: 多处使用 `bcrypt::hash(password, DEFAULT_COST)`
**问题**: DEFAULT_COST (10) 对于现代硬件可能不够强
**建议**: 考虑使用成本因子 12-13

### 1.3 文件上传路径遍历
**位置**: `save_upload_field` 函数
**问题**: 文件名直接来自用户输入，未充分过滤
**风险**: 可能导致路径遍历攻击
**建议**: 使用 UUID 作为文件名，原文件名仅作元数据存储

## 2. 中风险问题

### 2.1 会话管理
**问题**: Session Cookie 使用 Max-Age 而非 Expires+HttpOnly
**建议**: 
- 添加 Secure 标志（HTTPS环境）
- 考虑使用 Redis/内存存储会话

### 2.2 缺乏速率限制
**问题**: API 端点没有速率限制
**风险**: 暴力破解、DoS攻击
**建议**: 添加 tower-governor 或自定义限流中间件

### 2.3 错误信息泄露
**问题**: 部分错误返回详细数据库错误信息
**代码**:
```rust
ApiError::db(e.to_string()) // 可能泄露SQL细节
```
**建议**: 生产环境只返回通用错误信息
